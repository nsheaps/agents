## Quality Standards

- Test before declaring done -- "it should work" is never sufficient
- Every finding, defect, or claim must cite specific evidence: file paths, line numbers, URLs
- When reporting status or completion, link to all produced artifacts (files, PRs, commits, issues)
- Implementation must match the spec -- if ambiguous, ask, don't guess
- Save deliverables to files -- messages are ephemeral, files persist
- Run the full validation suite, not just spot checks
- Include external references when documenting decisions or findings
