# Trust Teammate Reports

Default posture: **trust what teammates report**. If something seems off, ask the teammate to back up their claim with evidence — do not escalate to the user.

## What TO Do

- Accept teammate reports at face value unless something contradicts known facts
- When skeptical, ask the teammate directly: "Can you show me evidence of X?" or "What led you to that conclusion?"
- Give teammates the opportunity to self-correct before involving the user
- Escalate to the user only when the teammate cannot resolve the ambiguity themselves

## What NOT to Do

- Second-guess teammate reports by immediately asking the user to verify
- Bypass the teammate and go straight to the user with "I'm not sure [teammate] is right"
- Silently ignore a report you doubt — surface the doubt to the teammate, not the user
- Treat teammate uncertainty as a reason to involve the user; let the teammate investigate first

## Escalation Path

```
teammate reports something → trust it
doubt arises → ask teammate for evidence
teammate provides evidence → resolved
teammate cannot resolve → THEN escalate to user
```

## Why This Matters

Unnecessary escalations to the user erode trust in the team and create interruptions. Teammates are capable of backing up their claims or correcting themselves when challenged directly. The user should only be involved when the team cannot resolve an issue internally.

## Applies To

- Orchestrators / team leads receiving status updates or findings from teammates
- Any agent evaluating another agent's reported results
- Decision points where a teammate's report influences the next action
