### Findings

**Potential issues (MEDIUM):**

1. **File**: `plugins/skill-required/hooks/scripts/cache-skill-read.sh:161-164`
   - **Severity**: Medium
   - **Description**: project_slug is generated by replacing `/` with `_`. This breaks for paths like `/home/user/my-project` → `home_user_my-project` (dash remains). Subsequent paths with dashes become ambiguous.
   - **Expected**: Should use base64 or hash the full path instead.
   - **Actual**: Lines 162-163 use sed: `echo "$project_dir" | sed 's|/|_|g' | sed 's|^_||'` — doesn't handle dashes.
   - **Impact**: Medium — cache may collide if project names have dashes in them. Unlikely but possible.
   - **Recommendation**: Use hash: `project_slug="$(echo "$project_dir" | md5sum | cut -d' ' -f1)"` or at least escape dashes too.

2. **File**: `plugins/skill-required/hooks/scripts/check-skill-required.sh:256-259`
   - **Severity**: Medium
   - **Description**: enabled flag parsing uses yq. If yq returns an unexpected value (not "true" or "false"), the check `[ "$enabled" = "false" ]` will allow all (line 258). Typos in config file are silently ignored.
   - **Expected**: Should validate that enabled is explicitly "true" or "false".
   - **Actual**: Line 257 uses default "true" if missing, but line 258 only checks for "false" explicitly.
   - **Recommendation**: Use explicit check: `[ "$enabled" = "true" ] && { allow or check rules }`

3. **File**: `plugins/skill-required/hooks/scripts/check-skill-required.sh:262-266`
   - **Severity**: Low
   - **Description**: skill_count is obtained via `yq '.skill-required.skills | length'`. If skills array is missing, yq returns "null" (string), not 0. The check `[ "$skill_count" = "0" ]` passes but `echo "0"` from yq might be "null".
   - **Expected**: Should convert null to 0 or check for both.
   - **Actual**: Line 263: `skill_count="$(yq ... 2>/dev/null || echo "0")"` — looks safe, but yq might return "null" string.
   - **Recommendation**: Add: `skill_count="${skill_count//null/0}"` to normalize.

4. **File**: `plugins/skill-required/hooks/scripts/check-skill-required.sh:337-338`
   - **Severity**: Low
   - **Description**: After increment, cache file is updated: `jq --argjson inc 1 '.tool_uses_since += $inc'`. If jq fails (bad JSON), mv overwrites with empty file.
   - **Expected**: Should check jq exit code before mv.
   - **Actual**: Line 338 pipes directly: `> "${cache_file}.tmp" && mv "${cache_file}.tmp" "$cache_file"` — okay, but could be safer with explicit error check.
   - **Recommendation**: Acceptable as-is (&& prevents mv if jq fails), but could be more explicit.

5. **File**: `plugins/skill-required/README.md:94-100`
   - **Severity**: Low
   - **Description**: Documentation says cache is at `~/.claude/cache/plugins/skill-required/<project-slug>/`. This implies cache survives across projects, but each project gets isolated cache dir. What if user switches projects? Cache persists but isn't cleaned.
   - **Expected**: Should document cache cleanup strategy or TTL.
   - **Actual**: No mention of when/how cache is invalidated.
   - **Recommendation**: Add note: "Cache persists for the session. To reset, delete `~/.claude/cache/plugins/skill-required/` or the specific project dir."
